# A Survey of Speculative Decoding Techniques
Source: blog.codingconfessions.com/p/a-selective-survey-of-speculative-decoding
Clipped: 2026-04-05

## Core Concept
Speculative decoding addresses a fundamental inefficiency: each LLM forward pass generates only one token despite GPU compute capacity remaining underutilized. It uses a smaller "draft" model to propose k candidates, then the base model verifies all k in one parallel forward pass.

## The Draft-Verify Mechanism
1. **Drafting phase**: speculator model generates k tokens sequentially from current context
2. **Verification phase**: base LLM processes context + k speculative tokens in a single forward pass
3. **Acceptance**: tokens accepted based on probability thresholds comparing draft vs. base distributions

### Acceptance Rule
"If the probability of the token as generated by the speculator is ≤ the probability of the same token as generated by the base LLM, then it is accepted."

### Medusa Entropy-Aware Acceptance
`p_original(x_{n+k}) > min(ε, δ * exp(-H(p_original)))`
- ε: hard threshold
- δ: scales with entropy — allows higher acceptance when generation is uncertain
- Accepts "most typical candidates" rather than requiring exact match

## Speedup Mathematics
- Best case: k tokens in the time of 1 base model pass
- Realistic: m accepted tokens (m < k) + verification overhead
- Empirical: Medusa-1 = 2.18x, Medusa-2 = 2.83x on Vicuna-7B
- Degrades with larger batch sizes (memory bandwidth saturation)

## Taxonomy of Methods

### 1. Independent Drafting (original, DeepMind 2022)
- Separate smaller model trained independently
- Requires alignment with base model output distribution
- Example: LaMDA-100M drafting for LaMDA-137B
- Problem: distribution mismatch hard to control

### 2. Self-Drafting
- Base model generates drafts at earlier layers or lower precision
- No separate model required
- Example: Medusa's extra prediction heads on same model

### 3. Medusa (2024, ICML)
- Multiple prediction heads attached to base model's final hidden layer
- Head i predicts token at position n+i from last hidden layer
- Top-k predictions from each head → tree of continuations for verification
- Medusa-1: 2.18x speedup, Medusa-2: 2.83x speedup
- Lightweight training: 5 hours on consumer GPU

### 4. EAGLE and Feature-Level Methods
- Extrapolates at feature level (penultimate layer hidden states) rather than token level
- Small auto-regressive head predicts next feature vectors
- Frozen classification head converts features → tokens
- Features are "more abstract and clearer" than token sequences → easier regression task

## Key Tradeoffs
| Factor | Tradeoff |
|--------|----------|
| Computation | More total compute; only worth it in memory-bound regimes |
| Accuracy | Entropy-aware acceptance balances diversity/quality |
| Latency vs throughput | Reduces per-token latency at batch=1; diminishes with batching |
| Training cost | Medusa heads: ~5h on consumer GPU; multi-stage variants: more |
| Hardware dependency | Only beneficial when memory bandwidth is the bottleneck |

## Critical Insight
Speculative decoding works by exploiting the autoregressive bottleneck through *speculative parallelism* — executing multiple forward passes assuming earlier predictions are correct, then verifying rapidly.
